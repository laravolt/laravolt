## P0 — Must fix before stable

### 1. Unify table/index UI around Laravolt core TableView + Suitable, not app-local Blade tables

**Problem**
VoltProcure index pages initially used raw Blade tables because the quickest MVP path did not expose an obvious, documented, polished default table pattern. That made the demo look inconsistent even though Laravolt already has Preline-styled table infrastructure.

**Source evidence**
- `src/Ui/TableView.php` renders `laravolt::ui-component.table-view.container`.
- `resources/views/ui-component/table-view/container.blade.php` already uses Preline/Tailwind card/table layout.
- `resources/views/ui-component/table-view/table.blade.php` already uses Preline-ish table classes.
- `resources/views/components/table.blade.php` exists but is a separate component with different API/classes.
- `packages/suitable/resources/views/table.blade.php` renders `<x-volt-table>`.
- Thunderclap generated index stubs already use `@livewire(...TableView::class)`.

**Expected outcome**
- One blessed table pattern for v7 docs and generated code:
  - Generated CRUD index page uses `Laravolt\Ui\TableView` + Suitable columns.
  - Table visuals are good by default: card container, header, spacing, hover states, empty state, search, filter, pagination, per-page.
  - Manual app docs discourage raw `<table>` for admin listings unless intentionally custom.

**Tasks**
- Audit and align:
  - `resources/views/components/table.blade.php`
  - `resources/views/ui-component/table-view/*.blade.php`
  - `packages/suitable/resources/views/table.blade.php`
  - `packages/suitable/resources/views/container.blade.php`
- Decide whether `<x-volt-table>` is the single low-level primitive and TableView/Suitable must render through it.
- Improve default table spacing/status/action styling to match Preline consistently.
- Add screenshots or browser assertions for generated index page visual structure.

**Acceptance criteria**
- A generated Thunderclap CRUD index looks acceptable without manual app-local table partials.
- Suitable and TableView share consistent Preline styling.
- VoltProcure can replace local Blade index tables with generated TableView components.

### 3. Permission cache hardening for database/file cache

**Problem**
VoltProcure hit:
`Laravolt\Platform\Models\User::permissions(): Return value must be of type Illuminate\Database\Eloquent\Collection, __PHP_Incomplete_Class returned`

The local demo workaround was `CACHE_STORE=array`, but stable Laravolt should tolerate normal cache stores and stale serialized values.

**Source evidence**
- `src/Platform/Concerns/HasRoleAndPermission.php::permissions()` caches `users.{id}.permissions` for one hour.
- With database cache, a cached serialized collection can return `__PHP_Incomplete_Class` and violate the return type.
- `AccessControlInvalidator` only forgets expected keys; it cannot protect against stale/corrupt/incompatible serialized values.

**Expected outcome**
Permission cache must be safe across deployment, package updates, class moves, cache stores, and app restarts.

**Tasks**
- Prefer caching primitive permission names/ids instead of Eloquent Collection objects, or validate cache value before returning.
- If invalid cache value is detected, forget key and rebuild.
- Add regression test for malformed/incomplete permission cache.
- Document recommended cache store for local/demo, but do not require `array` for production.

**Acceptance criteria**
- `hasPermission()` never throws TypeError because of cache value shape.
- Database cache works in demo app.
- Force-logout/access-control invalidation behavior remains intact.

### 4. Docker-first demo/deployment bundle for Laravolt apps

**Problem**
Dalang deployment without Docker was blocked by missing runtime tools. Dockerized demo makes Laravolt easier to try and gives a reusable release artifact.

**Current VoltProcure work**
- Added dev and prod Docker paths in VoltProcure:
  - `docker/php/dev.Dockerfile`
  - `docker/frankenphp/prod.Dockerfile`
  - `docker-compose.dev.yml`
  - `docker-compose.prod.yml`
  - `DOCKER.md`
- Prod target uses FrankenPHP + PHP 8.4, baked Composer deps, baked Vite assets, SQLite volume.

**Tasks for Laravolt v7**
- Decide if Laravolt starter kit should ship official Docker templates.
- Split dev vs production images cleanly.
- Support SQLite quickstart and PostgreSQL production compose.
- Add Octane/FrankenPHP guidance and safe defaults.
- Add CI build for Docker image.

**Acceptance criteria**
- New Laravolt v7 app can be run with `docker compose -f docker-compose.dev.yml up --build`.
- Production demo can be run with `docker compose -f docker-compose.prod.yml up --build -d`.
- Docs explain image targets and env vars.
